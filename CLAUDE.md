# CLAUDE.md — 久益检验报告更新器 项目上下文

## 项目概述

为宁波久益电子有限公司开发的**工作流自动化工具**，用于替代人工逐个修改检验报告的重复劳动。最终形态为一个轻量级 Windows `.exe` 程序，使用者为认知水平为普通工人的工厂员工。

## 业务工作流

原始人工流程：
1. 打开「发货计划」Excel 表格
2. 根据每行的「料件编号」（主码），在检验报告文件库中找到对应的检验报告
3. 将检验报告中的「出货数量」改为发货计划中对应的「本周要求数量」
4. 将检验报告中的「检验日期」改为当天日期
5. 「检验数量」字段不需要手动改——表格内已有与「出货数量」的公式联动关系
6. 逐份打印所有修改过的检验报告

## 数据结构

### 送货单 (.xlsx)（v2.0 新格式）
- **文件名格式**: `送货单-XXX+(N).xlsx`
- **行 1-7**: 表头（公司信息、收货地址等）
- **行 7**: 列标题（序号, 产品料号, 规格型号, 单位, 数量, 备注）
- **行 8+**: 数据行
- **列 B (index 1)**: `产品料号` — 如 `YY60038563`，作为主键匹配报告文件
- **列 E (index 4)**: `数量` — 整数，映射到报告的「出货数量」
- 末尾有合计行（产品料号为空），需跳过
- 支持一次导入多个送货单文件，合并处理
- 同一个料件编号可能跨文件重复出现，需要用户决定合并或分开处理

### 检验报告 (.xls，旧版 Excel 格式)
- **文件名格式**: `{客户料号} {久益料号} 出货检验报告.xls`，如 `YY60010071 J00016058 出货检验报告.xls`
- **匹配方式**: 文件名以「料件编号」开头
- **需修改的单元格**（COM 1-indexed）:
  - `Cells(2, 14)` → 出货数量（设为送货单中的数量）
  - `Cells(3, 14)` → 检验日期（含 `=TODAY()` 公式，自动更新，无需手动写入）
- **不修改的单元格**:
  - `Cells(2, 7)` = 检验数量 — 与出货数量有公式/函数关系，自动联动
- 报告含大量合并单元格和复杂格式，必须完整保留

### 报告库样例统计
- 样例送货单: 4 个文件，每个 3-24 条数据
- 样例报告库: 65 个 .xls 文件

## 技术方案

### 核心决策：VBScript + COM（不用 pywin32）
- Python 动态生成 VBScript 脚本
- 通过 `cscript.exe` 执行 COM 自动化操作 WPS/Excel
- **优势**: 完美保留公式和格式、无 pywin32 依赖、exe 体积更小
- COM ProgID 尝试顺序: `Excel.Application` → `KET.Application` → `ET.Application`

### 技术栈
- **语言**: Python 3.8+
- **GUI**: tkinter + ttk（内置，零依赖）
- **Excel 读取**: openpyxl（读 .xlsx 发货计划）
- **报告修改**: 生成 VBScript → cscript.exe → COM 操作 WPS
- **打印**: 通过 COM `wb.PrintOut()` 调用 WPS 打印
- **打包**: PyInstaller `--onefile --windowed`

### 文件结构
```
main.py           — 完整单文件应用（~350 行）
requirements.txt  — openpyxl, pyinstaller
build.bat         — Windows 构建脚本（纯 ASCII，避免编码问题）
出货检验报告/      — 检验报告 .xls 模板库（样例数据）
发货计划样例/      — 发货计划 .xlsx 样例
```

## 目标环境
- **操作系统**: Windows
- **办公软件**: WPS Office（非 Microsoft Excel）
- **用户**: 工厂普通工人，需要极简操作界面
- **分发**: 单个 .exe 文件，双击即用

## 重复料件处理逻辑

当同一个料件编号在发货计划中出现多次时：
1. **弹窗让用户选择**: 合并（求和） / 分开处理 / 跳过
2. **合并**: 数量累加，生成一份报告
3. **分开**: 打印时逐个设置不同数量后分别打印，文件最终保留最后一个值
4. 用户原话："最终还是随机选一个的数据覆盖保留在库中"

## 构建注意事项

- `build.bat` 中**不能包含中文字符**——Windows cmd 编码问题会导致乱码和构建失败
- 构建输出到 `C:\jiuyi_build\dist\ReportUpdater.exe`（纯 ASCII 路径）
- 项目路径可能包含中文，因此 PyInstaller 的 `--distpath`/`--workpath`/`--specpath` 必须指向无中文的路径
- `.bat` 文件的 `chcp 65001` 对批处理文件自身内容不可靠

## 开发环境 vs 运行环境

- **开发**: macOS — 可测试 GUI、数据读取、VBS 生成；VBS 执行部分会报错（无 cscript）
- **运行**: Windows + WPS — 全功能可用
- `run_vbs_script()` 中编码自适应：Windows 用 `mbcs`，其他平台用 `utf-8`

## GUI 设计原则

- 大字体（Microsoft YaHei）、大按钮、中文界面
- 两步操作：「① 更新报告」→「② 一键打印」
- 实时进度条 + 滚动日志区域
- 所有耗时操作在后台线程执行，GUI 保持响应
- VBS 通过 stdout 管道输出进度，Python 实时解析更新 UI
